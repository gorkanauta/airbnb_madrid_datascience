    Propuesta de Despliegue y Productivización de la Solución ✨

    En esta propuesta, se describe cómo lanzar el front y el back de la solución utilizando Flask y Streamlit.

    Primero! 1️⃣

    Lanzar el devcontainer en VSCode (ver seccion de configuracion de esta misma documentación).

    Backend 🖥️

    Para lanzar el backend de la solución, se utiliza el siguiente comando (en una terminal nueva):

    poetry run flask run

    El comando anterior ejecuta la aplicación Flask que se ha desarrollado. Flask es un framework ligero de Python utilizado para construir aplicaciones web. Al ejecutar este comando, el servidor web se inicia y la aplicación backend estará disponible en la dirección proporcionada por Flask.

    Frontend 🌐

    Para lanzar el frontend de la solución, se utiliza el siguiente comando (en una terminal nueva):

    poetry run streamlit run /workspaces/keepler_technical_assessment_gorka_bengochea/app/frontend/app.py

    El comando anterior ejecuta la aplicación Streamlit que se ha desarrollado. Streamlit es un framework de Python para crear aplicaciones web interactivas. Al ejecutar este comando, se iniciará el servidor de Streamlit y la aplicación frontend estará disponible en el navegador web en la dirección proporcionada por Streamlit.

    Descripción de la Solución 📝

    La solución se ha desarrollado utilizando Flask y Streamlit para crear una simulación de despliegue del modelo. Flask se utiliza para construir el backend, que proporciona la lógica de la aplicación y los puntos finales de la API. Streamlit se utiliza para construir el frontend, que presenta una interfaz interactiva para interactuar con el modelo y visualizar los resultados.

    Arquitectura de la Solución 🏗️

    La arquitectura de la solución se compone de dos partes:

    1. Backend: Implementado con Flask, el backend se encarga de recibir las solicitudes del frontend, procesarlas y devolver los resultados correspondientes. Se utiliza para la lógica del modelo y la comunicación con otras dependencias.

    2. Frontend: Implementado con Streamlit, el frontend proporciona una interfaz de usuario intuitiva para interactuar con el modelo. Permite enviar solicitudes al backend y visualizar los resultados de manera clara y comprensible.

    La combinación de Flask y Streamlit permite crear una solución completa y escalable para desplegar y productivizar el modelo.
